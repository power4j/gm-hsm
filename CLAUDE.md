## 文档规范

- 必须：文档使用 markdown 格式
- 必须：README.md 保持简洁，详细文档放到docs目录下,并根据用途建立子文件夹
- 代码注释风格必须严谨
  - 必须：使用英文进行注释,注释风格参考rust 社区最佳实践
  - 必须：禁止尾注释
  - 必须：禁止表情符号
  - 必须：禁止'记笔记'，'解答问题'等污染代码的行为

## 日志规范

- 必须：禁止在日志输出中使用emoji表情符号,可以使用 ASCII 表情符号 

## git 规范

- 应该：提交代码时，按照相关性分多次提交

## 其他

- 必须：临时性的文件都必须放在项目的tmp 目录下,并根据用途建立子文件夹
